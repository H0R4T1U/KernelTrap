Features

Everything you need to
detect and contain threats

A complete intrusion detection pipeline from kernel-level collection to automated attacker containment.

⚡

eBPF Syscall Monitoring

Uses Tracee (eBPF) or auditd to collect every syscall event from the Linux kernel — zero overhead, no kernel patches required.

🤖

ML Anomaly Detection

Isolation Forest model trained on the BETH dataset scores each event in real time. Detects behavioral anomalies, not just known signatures.

📷

Honeypot Pivot

After sustained suspicious activity, the user is transparently redirected into an isolated Docker honeypot via SSH — without alerting the attacker.

📊

BETH Dataset Format

All events are transformed to a standardized 16-column CSV format compatible with the BETH dataset for model training and evaluation.

🔒

Per-user Tracking

Anomaly windows are tracked per user with configurable thresholds. System users are automatically whitelisted to avoid false positives.

⚙

Systemd Integration

Ships with a systemd service for continuous background logging. Configure once, monitor forever with journald log integration.

How it works

From syscall to honeypot
in milliseconds

KernelTrap operates in a continuous pipeline — collecting, scoring, and acting on kernel events as they happen.

01

Collect syscall events via eBPF

Tracee attaches eBPF probes to the Linux kernel and emits a JSON stream of syscall events — including process ID, user ID, syscall name, arguments, and return values. No application changes needed. Supports tracee and auditd.

02

Transform to BETH format

syscall_logger.py parses each event and normalizes it into the 16-column BETH dataset format, extracting features like mountNamespace, parentProcessId, and returnValue used by the model.

03

Score with Isolation Forest

Each event is scored in real time by the trained Isolation Forest model. Events are classified into three severity levels: NORMAL, LOW, and HIGH — based on how anomalous the 7-feature vector is.

04

Track anomaly windows per user

A sliding time window tracks how many HIGH-severity events a user generates. If the threshold is exceeded (default: 5 events in 60 seconds), an automated pivot is triggered.

05

Pivot attacker to honeypot

The hp_pivot_user script sends SIGUSR1 to the user's shell, which transparently exec's into an SSH session pointing to the Docker honeypot on localhost:2222. The attacker continues — now in an isolated environment.
